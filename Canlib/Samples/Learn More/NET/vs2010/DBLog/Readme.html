A database logger

Source code

This program listens for messages on a channel, uses a database for interpreting those messages and writes the physical values of the signals to a .csv file.

This is a direct continuation of the Logger example. It also borrows a lot from part 7 in the tutorial.

Usage

Enter a channel number and press "Initialize". Then, select an output file and a database file. Choose a message from the drop-down menu and press "Load" and the "Start logging". The program will listen to any incoming messages matching the identifier of the database message, interpret them accordingly and print the physical values to the screen as well to the selected output file.

Interpreting the data

Just as in the previous example, we create a CSV header when we start and a CSV record for each incoming message. The difference is that the CSV header contains the names of all the signals in the message and the records contain the physical values for each signal.

Getting the names of all the signals is done by iterating through all the signals of the message handle and calling GetSignalName on each of them. The iteration is done by first calling GetFirstSignal and then GetNextSignal as long as these calls return the OK status. When there is no next signal, these calls will return the error status vaDbErr_NoSignal.

Getting the signal values is done by iterating through the signals as described above and using GetSignalValueFloat with the message data as parameter. This will output a double value with the signal's physical value.
