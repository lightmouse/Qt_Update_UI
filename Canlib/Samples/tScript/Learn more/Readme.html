Sample t scripts

This directory contains a few examples of how to use the t programming language.

EnvVar

A C program which demonstrates how to load a t script to a device as well as how to run it and influence its environment variables.

Other t programs

A collection of various scripts. Note that most of these programs don't put the channel on bus. You can do that either by using some program on your computer (like CanKing) or by running another t program which goes on bus.

Most of these programs have more detailed instructions in their comments.

bus_ctrl

Goes on bus with two channels on start, closes them on stop. The program has an environment variable for the bitrate. When the environment variable is updated, it goes off bus, updates the bitrate and goes on bus again.

default_can

Prints the id of any incoming CAN message without a more specific hook.

all_can

Demonstrates the [*] message hook. Prints the timestamp of the incoming messages and keeps track of the two consecutive messages with the smallest time between them.

can_calc

Should be used with the sample database. It waits for a message with an id matching the Engine message from the database. If the sequence number (one of the signals) of the message is in order, it will recalculate the values and create a new message which is transmitted. It also stores the incoming signal values in environment variables so they can be read by another program.

remote_response

Waits for a remote message with a certain identifier, then replies with a message whose data is stored in an environment variable.

script_ctrl

Demonstrates usage of environment variables and how to load scripts on other slots. The environment variables makes it possible to choose which channel and slot a program should use. Setting the environment variable for the file name will load the program and start it.

logger_ctrl

Starts or stops logging on the device depending on data from incoming messages.

timer_array

Responds to incoming messages one second after receiving them. The script can handle several delays at once using an array of timers.

send_list

Sends messages according to a list stored in a file.

sine_wave

Using the database, sends out a series of messages representing a sine wave.

osc_data

Reads and writes data to files, using the database to convert physical values to CAN messages.

exc_dump

Contains an example of how to handle exceptions.

licence

Shows how to create and handle devices for devices.

customer_data

Reads the customer data on the device (provided it exists) and prints it. Also demonstrates how to use the on key hook.
